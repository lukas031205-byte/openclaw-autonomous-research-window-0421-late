## OpenClaw systemd Issue

- `openclaw gateway install` uses a `--user`-level systemd service, which is buggy in this VM environment (version 3.2).

- KAS worked around it by manually creating a system-level systemd service.

- If the same issue happens again, create `/etc/systemd/system/openclaw.service` directly instead of using `openclaw gateway install`.
